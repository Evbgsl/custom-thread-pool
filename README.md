# Custom Thread Pool (Java)

## Описание проекта

В рамках курсовой работы реализован собственный пул потоков (`CustomThreadPool`) без использования `ThreadPoolExecutor`.

Цель проекта: продемонстрировать принципы многопоточного программирования, управления задачами, балансировки нагрузки и обработки перегрузок.

Реализация включает:

* настраиваемый пул потоков;
* распределение задач между потоками;
* масштабирование пула;
* обработку перегрузки (rejection policy);
* демонстрационную программу.

---

## Основные возможности

### 1. Настройка пула потоков

Пул поддерживает следующие параметры:

* `corePoolSize` - базовое количество потоков;
* `maxPoolSize` - максимальное количество потоков;
* `keepAliveTime` - время жизни неактивного потока;
* `queueSize` - размер очереди задач;
* `minSpareThreads` - минимальное количество свободных потоков.

---

### 2. Распределение задач (Round Robin)

Каждый поток (`Worker`) имеет собственную очередь задач.

Распределение задач реализовано по алгоритму **Round Robin**:

```java
int index = Math.abs(nextWorker.getAndIncrement() % workers.size());
```
Использование `AtomicInteger` гарантирует потокобезопасное распределение задач между очередями.

Это обеспечивает:
* равномерное распределение нагрузки;
* отсутствие перегрузки одной очереди;
* предсказуемое поведение системы.

---

### 3. Масштабирование пула

Пул динамически изменяет количество потоков:

* создаёт новые потоки при росте нагрузки;
* не превышает `maxPoolSize`;
* завершает лишние потоки при простое (`keepAliveTime`);
* поддерживает минимум свободных потоков (`minSpareThreads`).

Решение о создании нового потока принимается в случае, если:

* задача не может быть помещена в очередь текущего Worker;
* текущее количество потоков меньше `maxPoolSize`.

В этом случае создаётся новый Worker и задача направляется в его очередь.

---

### 4. Обработка отказов (Rejection Policy)

При перегрузке системы реализованы две стратегии:

#### AbortPolicy

* задача отклоняется;
* выбрасывается исключение `RejectedExecutionException`.

**Преимущества:**

* простота;
* явная сигнализация перегрузки.

**Недостатки:**

* возможная потеря задач.

---

#### CallerRunsPolicy

* задача выполняется в потоке вызывающего кода.

**Преимущества:**

* задачи не теряются;
* система автоматически замедляется.

**Недостатки:**

* блокируется вызывающий поток;
* увеличивается latency.

---

### 5. Поддержка Runnable и Callable

Пул реализует интерфейс:

```java
interface CustomExecutor extends Executor {
    void execute(Runnable command);
    <T> Future<T> submit(Callable<T> callable);
    void shutdown();
    void shutdownNow();
}
```

Поддерживается:

* выполнение `Runnable`;
* выполнение `Callable` с получением результата (`Future`);
* мягкое завершение (`shutdown`);
* немедленное завершение (`shutdownNow`).

---

### 6. Логирование

Система логирует ключевые события:

* создание потоков
* выполнение задач
* распределение задач по очередям
* отказ в обработке
* завершение потоков
* idle timeout

Пример:

```
[ThreadFactory] Creating new thread: MyPool-worker-1
[Pool] Task accepted into queue #0: Task-1
[Worker] MyPool-worker-1 executes Task-1
[Rejected] Queue is overloaded. Running task in caller thread: Task-6
```

## Качественный анализ производительности

### Сравнение с ThreadPoolExecutor

| Характеристика         | CustomThreadPool | ThreadPoolExecutor |
| ---------------------- | ---------------- | ------------------ |
| Гибкость настройки     | Средняя          | Высокая            |
| Контроль распределения | Полный           | Ограниченный       |
| Производительность     | Ниже             | Выше               |
| Простота использования | Ниже             | Выше               |

В данной реализации используется модель "один Worker - одна очередь", что упрощает контроль распределения задач, но уступает по эффективности централизованным и оптимизированным решениям стандартной библиотеки.

---

### Влияние параметров

* Малый `queueSize` → быстрее достигается перегрузка;
* Большой `maxPoolSize` → выше параллелизм, но больше overhead;
* Малый `keepAliveTime` → агрессивное освобождение ресурсов;
* `minSpareThreads` → уменьшает задержки при всплесках нагрузки;

---

## Архитектура

Основные компоненты:

* `CustomThreadPool` - управление пулом;

* `CustomThreadFactory` - создание потоков;
* `RejectionPolicy` - обработка перегрузки;
* `DemoTask` - демонстрационная задача.
